Create `docs/` directory and write `docs/email-setup.md` following the strict 11-step dependency chain from research. The guide must be a step-by-step walkthrough that someone can follow from start to finish.

Structure the guide in these sequential sections:

**Prerequisites section:**
- Cloudflare account with rajivrago.com domain
- Resend account (already exists -- same account used for contact-notify)
- Gmail account (use placeholder `your-email@gmail.com` throughout -- repo is public)

**Part 1: Cloudflare Email Routing (EMAIL-01)**
Steps to enable Email Routing in Cloudflare dashboard:
- Navigate to Email > Email Routing in Cloudflare dashboard
- Delete any pre-existing MX records if prompted
- Add routing rule: mail@rajivrago.com forwards to your-email@gmail.com
- Verify destination email (click link in Gmail)
- Confirm Cloudflare auto-configures MX records
- Verification command: `dig MX rajivrago.com +short` (expect route1/2/3.mx.cloudflare.net)
- Test: send email from external account to mail@rajivrago.com, confirm arrival in Gmail
- Troubleshooting: check Email Routing activity log if email doesn't arrive, verify all email DNS records are "DNS only" (gray cloud, not orange)

**Part 2: Resend Domain Verification (EMAIL-03 partial)**
Steps to verify rajivrago.com in Resend dashboard:
- Navigate to Resend dashboard > Domains > Add Domain
- Enter rajivrago.com
- Copy the DNS records Resend provides (SPF on `send` subdomain, DKIM on `resend._domainkey`, MX on `send` subdomain)
- Add each record in Cloudflare DNS dashboard -- all must be "DNS only" (gray cloud)
- Wait 10+ minutes for DNS propagation
- Verify with dig commands before clicking Verify in Resend:
  - `dig TXT send.rajivrago.com +short`
  - `dig TXT resend._domainkey.rajivrago.com +short`
  - `dig MX send.rajivrago.com +short`
- Click Verify in Resend dashboard
- Troubleshooting: if dig shows expected values but Resend says pending, wait longer; ensure records are not proxied

**Part 3: DMARC Record (EMAIL-03 partial)**
- Add TXT record in Cloudflare DNS: Name `_dmarc`, Value `v=DMARC1; p=none; rua=mailto:mail@rajivrago.com`
- Verify: `dig TXT _dmarc.rajivrago.com +short`
- Note: p=none is monitoring-only; tighten to p=quarantine after verifying deliverability

**Part 4: Edge Function Sender Update**
- Set SENDER_EMAIL secret in Supabase dashboard: Edge Functions > Secrets > add `SENDER_EMAIL` = `noreply@rajivrago.com`
- Note: the code change is handled separately (Task 2 of this plan), but the secret must be set in the dashboard

**Part 5: Gmail "Send Mail As" (EMAIL-02)**
- Navigate to Gmail Settings > Accounts and Import > Send mail as > Add another email address
- Enter name and mail@rajivrago.com
- Uncheck "Treat as alias" per user decision
- SMTP server: smtp.resend.com, Port: 587, Username: resend, Password: (Resend API key)
- Click Send Verification -- verification email routes through Cloudflare Email Routing to Gmail
- Enter verification code from the received email
- Set mail@rajivrago.com as default sender
- Enable "Reply from the same address the message was sent to"

**Part 6: Verification Checklist**
- Complete DNS verification commands (all 5 dig commands)
- Inbound test: external email to mail@rajivrago.com arrives in Gmail
- Outbound test: send from Gmail as mail@rajivrago.com to external address
- Authentication test: recipient checks "Show original" in Gmail -- SPF, DKIM, DMARC all pass

**Important notes for the guide:**
- Use placeholder `your-email@gmail.com` for the Gmail forwarding destination (repo is public)
- Include the SPF note from research: Resend's SPF goes on `send` subdomain, NOT root domain. No merge needed with Cloudflare's root SPF. However, note that if Resend dashboard shows root-domain records, dashboard values take precedence.
- Warn about DNS propagation delays throughout
- Include all dig/nslookup diagnostic commands inline at each verification step
