Follow `docs/email-setup.md` from start to finish. The guide has 5 parts that must be done in order due to the dependency chain:

1. **Cloudflare Email Routing** -- Enable Email Routing in Cloudflare dashboard. Add routing rule: mail@rajivrago.com forwards to your Gmail. Verify destination email. Run `dig MX rajivrago.com +short` (expect Cloudflare MX servers). Test: send email from external account to mail@rajivrago.com, confirm arrival in Gmail.

2. **Resend Domain Verification** -- Add rajivrago.com in Resend dashboard. Copy DNS records to Cloudflare (all "DNS only" / gray cloud). Wait 10+ minutes for propagation. Run `dig TXT send.rajivrago.com +short` and `dig TXT resend._domainkey.rajivrago.com +short`. Click Verify in Resend dashboard.

3. **DMARC Record** -- Add TXT record in Cloudflare: Name `_dmarc`, Value `v=DMARC1; p=none; rua=mailto:mail@rajivrago.com`. Verify with `dig TXT _dmarc.rajivrago.com +short`.

4. **Supabase Secret** -- Add `SENDER_EMAIL` = `noreply@rajivrago.com` in Supabase Edge Function secrets dashboard.

5. **Gmail "Send Mail As"** -- Add mail@rajivrago.com in Gmail Settings > Accounts > Send mail as. SMTP: smtp.resend.com, port 587, username: resend, password: Resend API key. Uncheck "Treat as alias". Complete email verification. Set as default. Enable "Reply from same address."
