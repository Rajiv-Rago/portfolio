    - stagger-item class sets opacity: 0 and transform: translateY(24px) by default
    - stagger-item.is-visible applies fade-up animation with animation-delay from --stagger-index CSS variable (calc(var(--stagger-index, 0) * 80ms))
    - animated-underline class creates a ::after pseudo-element underline that scales from 0 to 1 on hover (scaleX transform, origin switches from bottom-right to bottom-left)
    - prefers-reduced-motion: reduce sets stagger-item to opacity: 1 and transform: none, disables animation
    - prefers-reduced-motion: reduce disables animated-underline transition
    - Test fixtures provide makeProfile, makeProject, makeBlogPost, makeExperience factory functions matching types.ts interfaces
  </behavior>
  <action>
    1. Add to src/index.css (after the existing .scroll-reveal rules, before the @keyframes shimmer):

    ```css
    .stagger-item {
      opacity: 0;
      transform: translateY(24px);
    }

    .stagger-item.is-visible {
      animation: fade-up 0.6s ease-out forwards;
      animation-delay: calc(var(--stagger-index, 0) * 80ms);
    }

    .animated-underline {
      position: relative;
      display: inline-block;
    }

    .animated-underline::after {
      content: '';
      position: absolute;
      width: 100%;
      height: 2px;
      bottom: -2px;
      left: 0;
      background-color: var(--color-accent);
      transform: scaleX(0);
      transform-origin: bottom right;
      transition: transform 0.25s ease-out;
    }

    .animated-underline:hover::after {
      transform: scaleX(1);
      transform-origin: bottom left;
    }
    ```

    Add reduced-motion overrides inside the existing `@media (prefers-reduced-motion: reduce)` block (or create a new block after the existing one for clarity):

